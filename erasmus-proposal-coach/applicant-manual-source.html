1. Purpose Of This Manual

This manual explains how to use the Erasmus+ Proposal Coach step by step. It is written for applicants who want to prepare a strong first draft quickly and then improve it into a submission-ready proposal.

The app supports these Erasmus+ actions: KA 122 Short-term mobility for school education, KA 152 Youth Exchange, KA 153 Mobility of Youth Workers, KA 210 Small-scale Partnership, and KA 220 Cooperation Partnership.

2. What The App Does

The Erasmus+ Proposal Coach helps applicants to:

  • choose the most suitable Erasmus+ application type
  • build a project from factual inputs instead of starting from an empty page
  • reuse saved projects, partner profiles, and lead organisation data
  • structure the budget and finance logic
  • generate a first draft locally or with OpenAI
  • review the draft through scores, consistency checks, evaluator-style feedback, and deadline reminders
  • search the official English Programme Guide 2026 directly inside the app
  • export the result as a Word-friendly file or printable PDF

3. Before You Start

Before using the app, prepare the following:

  • the intended application type if known
  • project title or working title
  • lead organisation and partner organisations
  • participant profile, country mix, and main theme
  • the problem or need the project responds to
  • draft objectives and main activities
  • budget assumptions and important deadlines

4. Applicant Roadmap

Core Block: Shape The Application

This is the main working block of the site. Move through it from top to bottom: first choose the action type, then complete the factual project fields, then finance, then drafting and review.

You Write These Blocks

  • application type, title, lead organisation, and partners
  • countries, participants, activity days, theme, and deadlines
  • problem, target group, European added value, objectives, activities, and methodology
  • finance assumptions, budget notes, and practical justifications

The App Or AI Writes These Blocks

  • project concept summary
  • relevance and needs analysis
  • objectives and learning outcomes narrative
  • activities, partner roles, impact, risks, and finance narrative

Recommended Workflow

Step 1: Start a project. Use Start New Project for a fresh application. Use Saved Projects to load an earlier draft. Use the Example Library if you want a fast starting structure.

Step 2: Confirm the right action. Choose the application type first. If you are not sure, use the Application Adviser and write a short concept summary. The adviser suggests whether the idea fits KA 122, KA 152, KA 153, KA 210, or KA 220.

Step 3: Work through Shape The Application. Fill in the lead organisation, partners, countries, participant profile, activity days, theme, and deadlines. These fields drive almost everything else in the app.

Step 4: Write the project logic. Complete the need, target group, European added value, objectives, learning outcomes, activities, methodology, preparation, inclusion, dissemination, sustainability, evaluation, and risks.

Step 5: Complete finance. For KA 122, KA 152, and KA 153, the app can help estimate category-based mobility budgets from assumptions such as travellers, staff, nights, and sessions. For KA 210 and KA 220, the app supports lump-sum logic and internal budget distribution notes.

Step 6: Generate the draft. Use Generate Draft to build a local first version. Use AI Write Draft if you want OpenAI to draft the narrative sections automatically from your input.

Step 7: Review and refine. Read the quality checklist, the KA-fit warning, the evaluator mode, the consistency notes, the finance advice, and the deadline tracker. Improve weak sections before exporting.

Step 8: Save and export. Save the project, build the submission pack, and export to Word or PDF when the text is ready for internal review or portal transfer.

5. Main Parts Of The App

Saved Projects

Use this to store multiple applications in the browser. A new project clears the working draft but keeps the saved library available.

Example Library

Examples help users start quickly. Most are realistic but fictional drafting examples. Folder-based examples are based on the user's own local project materials.

Partner Library And Lead Organisation

Save partner cards once and reuse them in later applications. The lead organisation can also be selected from saved or current organisation information.

Guide Summary And Green Help

The green guidance under the fields changes with the chosen application type. This helps the applicant answer from the right Erasmus+ perspective.

Finance And Budget Helper

The finance section supports mobility-style category budgets and partnership-style lump-sum logic. It also drafts a finance paragraph and gives warnings if the justifications are too thin.

Activity Programme Builder

This section generates a day-by-day programme table from the entered number of days and the activity list. Use Regenerate to rebuild it or Clear to empty the view.

Programme Guide 2026 Search

The app includes a deep search block for the official English Programme Guide 2026. Users can type their own search terms, review match snippets, and open the relevant official guide page directly. When the search field is empty, no result cards are shown.

Review Tools

The app includes score cards, consistency checks, budget advice, deadlines, evaluator-style insights, smart warnings, and a final submission pack.

6. How The AI Works

The app can call the OpenAI Responses API directly from the browser if the user pastes an API key into the AI section.

  • AI does not run automatically.
  • AI only writes when the user clicks AI Write Draft or a section rewrite button.
  • AI writes narrative sections, not the raw factual form fields.
  • The API key is not stored by the app.
  • Better input leads to better AI output.

7. Data Storage And Privacy

The app stores saved projects, working draft data, and partner library data in the browser's local storage. This means the data stays local to the browser being used.

If AI writing is used, the current project data is sent to OpenAI for that request. Without an API key, the app stays entirely local and uses built-in drafting logic.

8. Good Practice For Applicants

  • start with facts, not with polished prose
  • keep the action type consistent with the target group and activity logic
  • make each objective visible in the activity design
  • use the finance helper to explain why the budget is proportionate
  • treat generated text as a strong draft, not as a final submission without review
  • check all final wording against the official Erasmus+ Programme Guide and the current call documentation

9. Why Use This App Before The Official Erasmus+ Environment

This app is designed as a drafting and preparation space. It helps applicants think, structure, write, and review before they transfer their final answers into the official Erasmus+ application environment.

Practical advantages of this app

  • it is easier to think in one continuous workspace than in a formal submission interface
  • it allows applicants to save multiple projects locally and compare versions
  • it helps build partner profiles and reuse lead organisation and partner data
  • it gives action-specific guidance under the fields, which the official environment does not structure in the same way
  • it includes draft generation, AI-supported writing, evaluator-style review, finance support, and consistency checks in one place
  • it includes direct search support for the official English Programme Guide while drafting
  • it lets applicants improve the logic first, instead of writing directly inside a submission system

Best use of both environments

The strongest workflow is usually to use this app first for concept development, drafting, and quality improvement, and only then use the official Erasmus+ environment for the final validated application text, annexes, and submission steps.

10. Limitations

This tool helps structure, accelerate, and strengthen drafting, but it does not replace official Erasmus+ documentation, call texts, or final compliance checks. Applicants remain responsible for the accuracy of content, the final budget, eligibility, annexes, and portal submission.

11. Final Advice

Use the app in three rounds: first to structure the facts, second to generate and improve the text, and third to review and prepare the final submission pack. That sequence gives the best results.
